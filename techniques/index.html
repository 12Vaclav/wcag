Techniques for WCAG 2.2

Change Log

A list of new, removed or significantly updated techniques:

  1. July 15th 2020: Removed Flash techniques.
  2. Sept 12th 2019: Added Failure of Success Criterion 2.5.1 due to providing functionality via a path-based gesture without simple pointer alternative
  3. Sept 12th 2019: Added Failure due to inability to deactivate motion actuation
  4. Sept 3rd 2019: Added Failure of Success Criterion 1.4.12 due to clipped or overlapped content when text spacing is adjusted
  5. Sept 3rd 2019: Added Failure of Success Criterion 4.1.3 due to providing status messages that cannot be programmatically determined through role or properties
  6. Aug 6th 2019: Added Failure of Success Criterion 1.4.10 due to content disappearing and not being available when content has reflowed
  7. Jul 16th 2019: Added Providing single point activation for a control slider
  8. Jul 16th 2019: Added Providing controls to achieve the same result as path based or multipoint gestures
  9. Jul 16th 2019: Added Failure of Success Criterion 1.3.4 due to showing a message asking to reorient device
  10. Jul 16th 2019: Added Using a control to allow access to content in different orientations which is otherwise restricted
  11. Jul 10th 2019: Added Creating a two-color focus indicator to ensure sufficient contrast
  12. Jul 10th 2019: Added Failure of Success Criterion 2.1.4 due to implementing character key shortcuts that cannot be turned off or remapped
  13. Jul 10th 2019: Added Making content on focus or hover hoverable, dismissible, and persistent
  14. Jun 18th 2019: Added Using native controls to ensure functionality is triggered on the up-event
  15. Jun 11 2019: Added Failure of Success Criterion 2.5.6 due to interactions being limited to touch-only on touchscreen devices
  16. Jun 18th 2019: Added Provide conventional controls and an application setting for motion activated input
  17. May 25th 2019: Added Semantically identifying a font icon with role="img"
  18. Apr 26th 2019: Added Failure due to locking the orientation to landscape or portrait view
  19. Feb 26th 2019: Removed F52 from SC 3.2.1 (still attached to SC 3.2.5)
  20. Feb 26th 2019: Added Provide sufficient contrast at the boundaries between adjoining colors
  21. Jan 10th 2019: Added Using the CSS reduce-motion query to prevent motion
  22. Dec 14th 2018: Added Ensuring that a contrast ratio of 3:1 is provided for icons
  23. Nov 30th 2018: Added Using CSS width, max-width and flexbox to fit labels and inputs
  24. Nov 30th 2018: Added F95 of 1.4.13 due to content shown on hover not being hoverable
  25. Nov 13th 2018: Added C34 Using media queries to un-fixing sticky headers / footers
  26. Nov 13th 2018: Added C36 Allowing for text spacing override
  27. Nov 13th 2018: Added C37 Using CSS max-width and height to fit images
  28. Nov 30th 2018: Added G207 Ensuring that drag-and-drop actions can be cancelled
  29. Nov 30th 2018: Added F95 Failure of Success Criterion 1.4.13 due to content shown on hover not being hoverable
  30. Nov 30th 2018: Added F96 Failure of Success Criterion 2.5.3 due to "accessible name" not containing the visible label text
  31. Nov 30th 2018: Added C38 Using CSS width, max-width and flexbox to fit labels and inputs
  32. Dec 14th 2018: Added G207 Ensuring that a contrast ratio of 3:1 is provided for icons

For a more detailed view of recent changes to the informative documents see the github updates.
