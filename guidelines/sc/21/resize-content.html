Discussion of the issue is available in Issue 77 and Pull Request 120. To file comments on this proposal, please raise new issues for each discrete comment in GitHub.

Content can be zoomed to 400% without loss of content or functionality, and without requiring scrolling in the direction of text except for parts of the content which require two-dimensional layout for usage or meaning.

Examples of content which require two-dimensional layout are images, maps, diagrams, video, games, data tables, and interfaces where it is necessary to keep toolbars in view whilst manipulating content.
