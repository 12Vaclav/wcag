Target Size

AA

Proposed

This is a proposed Success Criterion that has not yet been formally approved by the Working Group. Discussion of the issue is available in Issue 60. To file comments on this proposal, please raise new issues for each discrete comment in GitHub.

LEVEL AA

The size of the target is at least 44 by 44 CSS pixels for pointer inputs except when:

  • Customizable: There is a mechanism is available to change the size of the target independent of the level of page magnification.
  • Equivalent Target: The target is available through an equivalent link or control on the same page that is at least 44 by 44 CSS pixels.
  • Essential: A particular presentation of target is essential to the information being conveyed.
  • In-Page Link: The target is a text link where the destination is on the same page.
  • Content Reflows: The target can overlap another target if it is in content that can be reflowed.
  • Cannot be Modified: the target is controlled by the user agent and cannot be modified.

LEVEL AAA

The size of the target is at least 44 by 44 CSS pixels for pointer inputs.
