Target Size

AA

Proposed

This is a proposed Success Criterion that has not yet been formally approved by the Working Group. Discussion of the issue is available in Issue 60. To file comments on this proposal, please raise new issues for each discrete comment in GitHub.

LEVEL AA

The size of the target at the default viewport size is at least 44 by 44 CSS pixels for pointer inputs except when:

  • a mechanism is available to change the size of the target
  • the target is available through an equivalent link or control that is at least 44 by 44 CSS pixels or is an in-page link
  • the target cannot be 44 by 44 CSS pixels due to mandatory layout
  • the target is part of a group of navigation links or embedded in a block of text that is at least 22 by 22 CSS pixels
  • there are no other targets within 32 CSS pixels both horizontally and vertically
  • the target is controlled by the user agent and cannot be modified

LEVEL AAA

The size of the target at the default viewport size is at least 44 by 44 CSS pixels for pointer inputs.
