Timeouts

A

Proposed

This is a proposed Success Criterion that has not yet been formally approved by the Working Group. Discussion of the issue is available in Issue 14 and Pull Request 116. To file comments on this proposal, please raise new issues for each discrete comment in GitHub.

Where data can be lost due to timeouts, users are warned at the start of a process about the length of inactivity that generates the timeout, unless the data is preserved for a minimum of a 24 hours of user inactivity.
