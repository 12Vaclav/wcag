Popup Visibility

AA

Proposed

This is a proposed Success Criterion that has not yet been formally approved by the Working Group. Discussion of the issue is available in Issue 75. To file comments on this proposal, please raise new issues for each discrete comment in GitHub.

The following are true when a popup becomes visible, except where the visual presentation of the popup is controlled by the user agent and is not modified by the author:

Trigger
The popup does not obscure any essential content within the trigger.
Hover
If the popup is triggered via pointer hover, then the pointer may be moved onto the popup without loss of popup visibility.
Focus
The popup remains visible while the trigger or any popup component has keyboard focus, unless the user takes an explicit action to close the popup.
