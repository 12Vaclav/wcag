Single-Character Key Shortcuts

A

Proposed

This is a proposed Success Criterion that has not yet been formally approved by the Working Group. Discussion of the issue is available in Issue 69 and Pull Request 133. To file comments on this proposal, please raise new issues for each discrete comment in GitHub.

If a single key shortcut using only printing characters is implemented by the web page to activate a control, then a mechanism is available to do one of the following:

  1. turn them off
  2. remap them to a shortcut that uses at least one non-printing key
  3. remaps them to a string of as many as 25 printing characters.
