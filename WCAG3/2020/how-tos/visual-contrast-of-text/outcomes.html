  • Select font characteristics and background colors to provide enough contrast. Also verify that both the font color and background is set. If the background is an image, affirm that the color is close to the image in case the image fails to load.
  • Use the APCA Contrast tool to test different pairs of colors.
  • Use of transparency will affect how colors render, so the CSS color values themselves will not predict contrast accurately. In these cases, using a digital color meter (eyedropper), inspect the color value as rendered and use that in the APCA tool.
  • Visually compare potential font selections against the "standard" fonts types listed in the visual contrast matrix, and note any perceived contrast difference for a given weight and size.
  • Test media queries to ensure different devices render the expected font size and weight. This is especially important if using units such as em or vw.
  • Take a screen shot of the content design, then use a CVD Simulator to see how that content appears to different types of color vision deficiencies (aka color blindness).

  • Use default fonts and colors but if defaults are to be used, the CSS cascade and inline styles must be checked to confirm that neither the font nor the background is set.
